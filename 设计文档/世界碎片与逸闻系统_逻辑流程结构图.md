# 世界碎片与逸闻系统_逻辑流程结构图

状态：草案  
关联文档：[世界碎片与逸闻系统.md](E:\PK\设计文档\世界碎片与逸闻系统.md)

本文档用于整理“世界碎片与逸闻系统”的系统逻辑流程。当前重点覆盖 4 个核心流程：

- 探索中触发文本
- 探索选择受文本影响
- 图鉴解锁
- 故事集收集

## 1. 系统总流程

```mermaid
flowchart TD
    A["玩家进入主动探索玩法"] --> B{"是否为挂机产出结算？"}
    B -->|"是"| B1["不触发世界碎片系统"]
    B -->|"否"| C["读取当前探索上下文"]

    C --> C1["地图信息<br/>map_id / map_tag"]
    C --> C2["事件信息<br/>event_id / trigger_type"]
    C --> C3["怪物与掉落信息<br/>monster_id / monster_tag"]
    C --> C4["玩家进度<br/>等级 / 主线 / 已获得记录"]

    C1 --> D["筛选可触发内容"]
    C2 --> D
    C3 --> D
    C4 --> D

    D --> E{"是否命中触发规则？"}
    E -->|"否"| E1["继续正常探索"]
    E -->|"是"| F["获得内容碎片<br/>lore_fragment"]

    F --> G{"该文本是否首次触发？"}
    G -->|"否"| G1["按重复规则处理<br/>不再触发 / 转化奖励 / 记录次数"]
    G -->|"是"| H["屏幕指定位置出现文本图标"]

    H --> I["飘字展示摘要或关键内容"]
    I --> J["自动记录已获得 / 已触发"]
    J --> K["执行关联效果<br/>lore_effect"]

    K --> L{"效果类型"}
    L -->|"探索线索"| M["写入探索线索状态"]
    L -->|"图鉴解锁"| N["更新图鉴归档"]
    L -->|"故事残页"| O["更新故事集进度"]
    L -->|"纯文本归档"| P["进入已读文本库"]

    M --> Q["后续探索出现相关选择或隐藏结果"]
    N --> R["图鉴页签红点 / NEW 提示"]
    O --> S["故事集页签红点 / 完成提示"]
    P --> T["玩家可在归档界面复看"]
```

## 2. 探索中触发文本

```mermaid
flowchart TD
    A["自动探险进行中"] --> B["触发事件"]
    B --> B1["发现尸体"]
    B --> B2["怪物掉落行囊"]
    B --> B3["获得残卷"]
    B --> B4["打开宝箱"]
    B --> B5["调查闪光杂物"]

    B1 --> C["查询 lore_trigger_config"]
    B2 --> C
    B3 --> C
    B4 --> C
    B5 --> C

    C --> D{"是否符合触发条件？"}
    D -->|"否"| D1["事件按普通规则结束"]
    D -->|"是"| E["读取内容碎片<br/>lore_fragment_config"]

    E --> F{"是否首次触发？"}
    F -->|"否"| F1["执行重复获得规则"]
    F -->|"是"| G["屏幕指定位置出现文本图标"]

    G --> H["根据 content_type 显示图标<br/>信件 / 书本 / 残卷 / 笔记"]
    H --> I["飘字逐渐展示摘要或关键内容"]
    I --> J["自动获得探索线索"]
    J --> K["自动执行关联效果"]

    K --> L{"是否关联收藏 / 图鉴 / 成就？"}
    L -->|"否"| M["仅进入已读文本库"]
    L -->|"是"| N["解锁对应进度并显示红点"]

    M --> O["返回自动探索流程"]
    N --> O
```

关键规则：

- 文本不需要玩家停下流程完整阅读。
- 首次触发时通过图标与飘字传达关键内容。
- 效果默认自动生效。
- 后续可在已读文本、图鉴或故事集中复看。

## 3. 探索选择受文本影响

```mermaid
flowchart TD
    A["玩家获得线索文本"] --> B["自动写入探索线索状态"]
    B --> C["继续探索"]
    C --> D["到达岔路 / 机关 / 资源点"]

    D --> E["读取当前探索节点"]
    E --> F{"是否存在可匹配线索？"}

    F -->|"否"| G["显示普通选择"]
    G --> G1["向左深入"]
    G --> G2["向右深入"]

    F -->|"是"| H["显示被线索影响的选择"]
    H --> H1["普通选项"]
    H --> H2["线索匹配选项<br/>显示信件图标 / NEW / 发现补给"]

    H2 --> I{"玩家是否选择线索匹配方向？"}
    I -->|"否"| I1["进入普通路线结果"]
    I -->|"是"| J["触发隐藏结果"]

    J --> J1["发现补给"]
    J --> J2["发现隐藏宝箱"]
    J --> J3["触发新事件"]
    J --> J4["更新地图或探索记录"]

    J1 --> K["继续探索"]
    J2 --> K
    J3 --> K
    J4 --> K
    I1 --> K
```

关键规则：

- 文本线索进入探索状态后，后续节点可以读取该状态。
- 线索匹配选项需要有明显 UI 提示，例如信件图标、NEW 角标、绿色收益提示。
- 玩家未选择线索方向时，仍可进入普通路线结果。
- 是否允许玩家未获得线索也发现隐藏结果，由具体地图节点配置决定。

## 4. 图鉴解锁

```mermaid
flowchart TD
    A["获得情报型内容碎片"] --> A1["猎人笔记 / 护卫记录 / 商人信件 / 怪物残页"]
    A1 --> B["首次获得后自动生效"]
    B --> C["读取 lore_effect_config"]

    C --> D{"effect_type 是否为图鉴解锁？"}
    D -->|"否"| D1["按其他效果处理"]
    D -->|"是"| E["读取 lore_archive_config"]

    E --> F{"图鉴条目是否已存在？"}
    F -->|"否"| G["新建图鉴条目"]
    F -->|"是"| H["追加图鉴新增信息"]

    G --> I["解锁条目：例如 狼王"]
    H --> I

    I --> J["写入图鉴内容"]
    J --> J1["基础描述"]
    J --> J2["特殊状态"]
    J --> J3["来源记录"]

    J2 --> K["例：重伤时会呼唤同伴"]
    J3 --> L["例：护卫队残留笔记"]

    K --> M["图鉴按钮显示红点 / NEW"]
    L --> M
    M --> N["玩家进入图鉴查看详情"]
```

关键规则：

- 图鉴信息由文本来源解锁，而不是只通过击杀解锁。
- 图鉴中的“应对倾向”应尽量包装为来源记录中的观察，而不是系统直接给攻略。
- 可支持同一图鉴条目多阶段解锁：基础信息、特殊状态、生态记录、弱点记录等。

## 5. 故事集收集

```mermaid
flowchart TD
    A["玩家进行主动探索 / 地图复刷"] --> B["触发故事残页掉落规则"]
    B --> C["读取 lore_trigger_config"]
    C --> D{"是否命中残页掉落？"}

    D -->|"否"| D1["继续探索"]
    D -->|"是"| E["获得故事残页"]

    E --> F["读取 lore_collection_config"]
    F --> G["写入故事集进度"]

    G --> H{"该页是否首次获得？"}
    H -->|"否"| H1["按重复获得规则处理"]
    H -->|"是"| I["显示获得提示"]

    I --> I1["例：获得 荆棘谷的青山 - 17页"]
    I --> J["逸闻录 / 故事集页签显示红点"]

    J --> K{"是否收齐故事集？"}
    K -->|"否"| L["更新已收集页数"]
    K -->|"是"| M["显示完成提示"]

    M --> M1["例：恭喜完成 荆棘谷的青山 收集"]
    M --> M2["故事标题显示 完 角标"]
    M --> M3["解锁完整故事文本"]

    L --> N["玩家可进入逸闻录分页阅读"]
    M3 --> N
    N --> O["分页查看：第 1 / 31 页等"]
```

关键规则：

- 故事残页只从主动探索、地图复刷、事件、宝箱、战斗掉落等主动玩法中获得。
- 不进入挂机产出。
- 故事集需要支持页码、总页数、完成状态、红点提示和完整故事查看。
- 收齐奖励是否包含数值奖励、称号、头像框或纯收藏成就，后续再定。

## 7. 逻辑优先级

1. 排除挂机产出。
2. 判断触发来源是否属于主动玩法。
3. 根据地图、事件、怪物、玩家进度筛选触发规则。
4. 判断是否首次触发。
5. 首次触发时显示图标与飘字。
6. 自动写入线索、图鉴、故事集或已读文本。
7. 对图鉴、逸闻录、收藏、成就等入口显示红点。
8. 后续探索节点读取线索状态，改变可选项或结果。
