- 支线任务：181条，按官方任务总表中“任务类型”为“支线”的条目筛选，包含本体、石之心、血与酒。
- 可读物/文本物件：1025个标题，合并书本、信件/笔记、告示板文本及两个资料片分类页，去重后统计。

## 对 PK 的直接参考点

- 支线不是单一规格：包含角色后续、地点小事件、连锁剧情、比赛/牌局、隐藏发现、任务链分支、资料片区域运营内容。
- 可读物承担四种功能：触发任务、补足环境背景、制造地方生活感、解释战利品或探索目标。
- 告示板是高效入口：同一界面同时承载委托、谣言、生活公告、地方冲突和轻喜剧文本。
- 信件/笔记常用于“事后叙事”：玩家看到结果，再通过文本倒推原因，成本低但沉浸感强。
- 同一主题会跨系统复用：一个地点事件可同时出现在告示、尸体信件、支线目标、日志记录和奖励配置中。

## 可转化为 PK 的内容结构

### 逸闻结构

| 字段 | 说明 |
| --- | --- |
| anecdote_id | 逸闻ID，用于记录、触发、任务链引用 |
| 标题 | 逸闻标题，可参考巫师3的短标题命名法 |
| region_id / location_id | 发生区域与地点，决定地方性文本风格 |
| trigger_type | 触发方式：告示、路过、拾取、NPC、战斗后、探索点 |
| trigger_condition | 等级、主线进度、时间、天气、阵营关系、前置事件 |
| content_type | 支线、纯文本、信件、谣言、日志、收藏记录 |
| followup_quest_id | 是否开启任务或后续追踪 |
| record_rule | 是否进入图鉴、日志、传闻录或地点档案 |

### 触发式文本结构

| 字段 | 说明 |
| --- | --- |
| text_id | 文本ID |
| text_group | 文本组，如地点闲谈、尸体信件、告示板、战斗后发现 |
| speaker_or_source | 来源：NPC、告示、信件作者、未知记录者、旁白系统 |
| display_channel | 弹窗、日志、物品详情、NPC对话、地图点描述 |
| priority | 同场景多文本触发时的优先级 |
| cooldown | 重复触发间隔 |
| archive_tag | 归档标签，如地区、势力、人物、事件 |
